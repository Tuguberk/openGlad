# Contributing to openGlad

First off, thank you for considering contributing to openGlad! It's people like you that make openGlad such a great tool for the community.

### 1. Where do I go from here?

If you've noticed a bug or have a question, search the [issue tracker](https://github.com/tugberkakbulut/openGlad/issues) to see if someone else in the community has already created a ticket. If not, go ahead and make one!

### 2. Fork & create a branch

If this is something you think you can fix, then fork openGlad and create a branch with a descriptive name.

### 3. Get the test suite running

Make sure you have Node.js 18+ installed.

```bash
cd openGlad
npm install
npm run dev
```

### 4. Implement your fix or feature

At this point, you're ready to make your changes. Feel free to ask for help; everyone is a beginner at first. Any improvements to the prompt templates, Mistral AI integration, or new tools for the Friction Engine are very welcome.

### 5. Create a Pull Request

When you're finished, push your changes to your fork and create a Pull Request against the main `openGlad` repository.

- Ensure the PR description clearly describes the problem and solution.
- Include the relevant issue number if applicable.

### Code of Conduct

By participating in this project, you are expected to uphold our standards of conduct. We are committed to providing a welcoming and inspiring community for all.
